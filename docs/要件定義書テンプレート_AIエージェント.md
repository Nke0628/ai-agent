### 3.2 ツール定義（エージェントが呼び出せる操作）

### 3.3 ナレッジ・データソース

### 3.4 会話・コンテキスト要件

| 項目 | 要件 |
|---|---|
| マルチターン対応 | 例：同一セッション内で文脈を保持する |

| 項目 | 要件 |
|---|---|
| 回答形式 | 例：日本語、Markdown、簡潔な敬体 |
| 根拠の提示 | 例：回答には必ず出典（文書名・リンク）を付与する |

## 5. 品質・評価要件（精度目標）

> 従来の「バグゼロ」ではなく、**評価セットに対する目標値と測定方法**で品質を定義する。

### 5.1 精度目標

| 評価項目 | 目標値 | 測定方法 |
|---|---|---|
| 回答正確性 | 例：評価セットで90%以上 | 人手評価＋LLM評価の併用 |
| ハルシネーション率 | 例：根拠のない回答 5%未満 | 出典との突合 |
| 意図分類の精度 | 例：95%以上 | ラベル付きテストデータ |
| 有害・不適切応答 | 例：0件 | レッドチーミング |

### 5.2 評価セット（ゴールデンデータセット）

| 項目 | 内容 |
|---|---|
| 件数・内訳 | 例：代表質問100件＋エッジケース30件＋攻撃的入力20件 |
| 作成者 | 例：業務部門がレビューし正解を確定する |
| 更新方針 | 例：本番の失敗事例を毎月追加する |

### 5.3 評価の運用

| 項目 | 内容 |
|---|---|
| リリース前評価 | 例：評価セット全件で目標値クリアをリリース条件とする |
| 継続評価 | 例：本番ログから週次でサンプリング評価 |
| ユーザーフィードバック | 例：回答ごとにGood/Badボタンを設置し集計 |

## 6. 安全性・ガードレール要件

### 6.1 入力ガードレール

| 項目 | 要件 |
|---|---|
| プロンプトインジェクション対策 | 例：外部文書由来のテキストは指示として扱わない設計とし、攻撃パターンを評価セットに含める |
| 不適切入力の処理 | 例：ハラスメント・無関係な依頼は定型応答で拒否する |
| 入力サイズ制限 | 例：1メッセージ○○文字まで |

### 6.2 出力ガードレール

| 項目 | 要件 |
|---|---|
| 禁止事項 | 例：法的助言・医療助言・人事評価に関する断定をしない |
| 機密情報の漏洩防止 | 例：ユーザーの権限外の文書は検索対象から除外する（後述のアクセス制御と連動） |
| 個人情報の取り扱い | 例：応答に他者の個人情報を含めない |

### 6.3 行動範囲の制限（エージェントの権限）

| 項目 | 要件 |
|---|---|
| ツール実行権限 | 例：読み取り系は自動、書き込み系は承認必須（3.2参照） |
| 実行回数・ループ制限 | 例：1リクエストあたりツール呼び出し○回まで |
| 課金・コスト暴走防止 | 例：ユーザーあたり日次トークン上限を設定 |

### 6.4 責任と透明性

| 項目 | 要件 |
|---|---|
| 免責・利用規約 | 例：初回利用時に注意事項へ同意を求める |
| 監査可能性 | 例：全ての判断（検索結果・ツール実行・最終応答）をログに記録する |

## 7. 非機能要件

### 7.1 性能

| 項目 | 要件 |
|---|---|
| 応答開始時間 | 例：ストリーミングで2秒以内に応答開始 |
| 応答完了時間 | 例：通常質問で15秒以内 |
| 同時利用 | 例：同時50セッション |

### 7.2 コスト

| 項目 | 要件 |
|---|---|
| API利用料の上限 | 例：月額○○円以内 |
| コスト監視 | 例：日次でダッシュボード確認、閾値超過でアラート |
| コスト最適化 | 例：プロンプトキャッシュ活用、簡易タスクは軽量モデルに振り分け |

### 7.3 セキュリティ・データ保護

| 項目 | 要件 |
|---|---|
| 認証・認可 | 例：社内SSOと連携。ユーザーの権限に応じて検索対象を制御する |
| データの外部送信 | 例：LLM APIへの送信データ範囲を定義し、学習利用されない契約形態を選択する |
| 個人情報 | 例：送信前にマスキング処理を行う |
| ログの取り扱い | 例：会話ログは暗号化保存、閲覧権限は運用管理者のみ |

### 7.4 可用性・運用

| 項目 | 要件 |
|---|---|
| 稼働時間 | 例：平日9:00〜18:00（業務時間内） |
| LLM API障害時 | 例：フォールバックモデルへ切替 / 「現在利用できません」を表示し従来窓口を案内 |
| 監視 | 例：エラー率・レイテンシ・トークン消費量・エスカレーション率を監視 |
| ナレッジ運用 | 例：回答不能だった質問を月次でレビューし、ナレッジに反映する体制を定義 |
