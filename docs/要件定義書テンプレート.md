## 6. 非機能要件

### 6.1 性能・拡張性

| 項目 | 要件 |
|---|---|
| レスポンスタイム | 例：通常画面遷移 3秒以内（95パーセンタイル） |
| 同時接続ユーザー数 | 例：最大100ユーザー |
| バッチ処理時間 | 例：日次バッチは6時間以内に完了 |
| 拡張性 | 例：ユーザー数2倍まで構成変更なしで対応可能 |

### 6.2 可用性

| 項目 | 要件 |
|---|---|
| 稼働時間 | 例：24時間365日（計画停止を除く） |
| 稼働率 | 例：99.5%以上 |
| 目標復旧時間（RTO） | 例：4時間 |
| 目標復旧時点（RPO） | 例：直前の日次バックアップ |

### 6.3 セキュリティ

| 項目 | 要件 |
|---|---|
| 認証 | 例：ID/パスワード + 多要素認証 |
| 認可 | 例：ロールベースのアクセス制御 |
| 通信の暗号化 | 例：TLS 1.2以上 |
| データの暗号化 | 例：個人情報は保存時に暗号化 |
| 監査ログ | 例：操作ログを1年間保管 |
| 脆弱性対応 | 例：リリース前に脆弱性診断を実施 |

### 6.4 運用・保守

| 項目 | 要件 |
|---|---|
| 監視 | 例：死活監視・リソース監視・エラー通知 |
| バックアップ | 例：日次フルバックアップ、保管期間30日 |
| メンテナンス | 例：計画停止は月1回、深夜帯に実施 |
| サポート体制 | 例：平日9:00〜18:00 |

### 6.5 移行要件

| 項目 | 要件 |
|---|---|
| データ移行 | 例：既存システムから顧客データを移行 |
| 移行方式 | 例：一括移行 / 段階移行 |
| 並行稼働 | 例：新旧システムを1か月間並行稼働 |

### 6.6 システム環境

| 項目 | 要件 |
|---|---|
| インフラ | 例：AWS（東京リージョン） |
| 対応ブラウザ / デバイス | 例：Chrome / Edge 最新版、スマートフォン対応 |
| 開発言語・フレームワーク | 例：TypeScript / Next.js（指定がある場合のみ） |
